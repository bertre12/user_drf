Используемые адреса:
	http://127.0.0.1:8000/api/login/
	http://127.0.0.1:8000/api/students/
	http://127.0.0.1:8000/api/students/{id}/

Методы:
	GET
	POST
	PUT
	PATCH

Коды ответов сервера:
	200 - Он свидетельствует о том, что обмен данными между клиентом и сервером прошли успешно.
	201 - Информирует об успешном создании нового ресурса в результате выполнения запроса.
	400 - Ошибка свидетельствует от том, что сервер не понял запрос пользователя из-за синтаксической ошибки.
	404 - Сообщает о том, что запрашиваемая страница не найдена.
	405 - Сообщает, что в запросе используется метод, который не поддерживается сервером.


----------------------------------------

Адрес 

	http://127.0.0.1:8000/api/login/

осуществляет вход на сервер, где происходит проверка имени и пароля. 
Используемый метод запроса - POST.

* При правильном запросе формы:

	{
	  "name": "string",
	  "password": "string"
	}

ответ сервера Code 200 и сообщение:

	{
	  "message": "Вы вошли как (имя пользователя)"
	}

* При неправильном запросе формы, когда введено некорректный пароль:

ответ сервера Code 400 и сообщение:

	Error: Bad Request
	{
	  "non_field_errors": [
		""Неверный пароль.""
	  ]
	}

* При неправильном запросе формы, когда введено некорректное имя пользователя:

ответ сервера Code 400 и сообщение:

	Error: Bad Request
	{
	  "non_field_errors": [
		"Такого пользователя не существует."
	  ]
	}

Адрес

	http://127.0.0.1:8000/api/students/

осуществляет регистрацию нового пользователя.
Используемый метод запроса - POST.

* При правильном запросе формы:

	{
	  "name": "string",
	  "password": "string",
	  "status": "Учится",
	  "level": 0
	}

ответ сервера Code 201 и сообщение:

	{
	  "name": "имя пользователя",
	  "password": "пароль",
	  "status": "статус пользователя",
	  "level": (выбранный уровень)
	}

* При неправильном запросе формы, когда когда введены некорректные данные в поле level:

ответ сервера Code 400 и сообщение:
	Error: Bad Request
	{
	  "level": [
		"Недопустимый первичный ключ \"(введённые данные)\" - объект не существует."
	  ]
	}

* При неправильном запросе формы, когда введены некорректные данные в поле status:

ответ сервера Code 400 и сообщение:

	Error: Bad Request
	{
	  "status": [
		"Значения (введённые данные) нет среди допустимых вариантов."
	  ]
	}

Адрес

http://127.0.0.1:8000/api/students/{id}/

осуществляет получение информации о пользователе, редактирования данных через {id} пользователя.
Используемый метод запроса - GET, PUT, PATCH.

1 - Метод GET.

* При запросе, с указанием действующим {id} пользователя:

ответ сервера Code 200 и сообщение:
	{
	  "name": "(имя пользователя)",
	  "password": "(пароль пользователя)",
	  "nickname_tg": null,
	  "nickname_inst": null
	}

* При запросе, без указания {id} пользователя:

ответ сервера Code 405 и сообщение:

	Error: Method Not Allowed

* При запросе, с неверным {id} пользователя:

ответ сервера Code 404 и сообщение:

	Error: Not Found
	{
	  "detail": "No Student matches the given query."
	}

2 - Метод PUT.

* При запросе формы, с действующим {id} пользователя:

	{
	  "password": "(новый пароль)",
	  "nickname_tg": "(новые данные)",
	  "nickname_inst": "(новые данные)"
	}

ответ сервера Code 200 и сообщение:

	{
	  "password": "(новый пароль)",
	  "nickname_tg": "(новые данные)",
	  "nickname_inst": "(новые данные)"
	}

* При запросе формы, с действующим {id} пользователя, но с неуказанными данными или пробелами в поле: password, nickname_tg, nickname_inst:

ответ сервера Code 400 и сообщение:

	Error: Bad Request
	{
	  "password": [
		"Это поле не может быть пустым."
	  ],
	  "nickname_tg": [
		"Это поле не может быть пустым."
	  ],
	  "nickname_inst": [
		"Это поле не может быть пустым."
	  ]
	}

3 - Метод - PATCH.

* При запросе формы, с действующим {id} пользователя:

	{
	  "password": "(новый пароль)",
	  "nickname_tg": "(новые данные)",
	  "nickname_inst": "(новые данные)"
	}

ответ сервера Code 200 и сообщение:

	{
	  "password": "(новый пароль)",
	  "nickname_tg": "(новые данные)",
	  "nickname_inst": "(новые данные)"
	}
	
* При запросе формы, с действующим {id} пользователя, но с неуказанными данными или пробелами в поле: password, nickname_tg, nickname_inst:

ответ сервера Code 400 и сообщение:

	Error: Bad Request
	{
	  "password": [
		"Это поле не может быть пустым."
	  ],
	  "nickname_tg": [
		"Это поле не может быть пустым."
	  ],
	  "nickname_inst": [
		"Это поле не может быть пустым."
	  ]
	}
